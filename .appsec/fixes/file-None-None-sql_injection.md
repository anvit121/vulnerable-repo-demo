# Security Fix — sql_injection

## Vulnerability
The original code is vulnerable to SQL injection because it directly concatenates user input into SQL queries, allowing attackers to manipulate query logic. Using parameterized queries prevents this by separating code from user input.

## Vulnerable code
cursor.execute('SELECT * FROM users WHERE name = '' + username + ''')

## Fixed code
cursor.execute('SELECT * FROM users WHERE name = %s', (username,))

## Why this works
The fix uses parameterized queries, passing the username as a parameter to the execute method, which prevents SQL injection by ensuring user input is treated as data, not code.
